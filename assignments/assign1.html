# Assignment 1 - Guessing Game created using Kotlin:

KotlinGuessingGameStarterProject.zip

Using Kotlin create a new console application titled Guessing Game. The application you create will be a number guessing game similar to the one you created last semester in MAD9021.

The game will generate a random number then the user will attempt to guess the random number. With each guess an appropriate type of integer variable will increment by one, this variable will keep track of the number of guesses. The user will be receiving guidance from the program in the form of text statements indicating if the guess is too high or too low. Once the number has been guessed the program will display a congratulations message and display the number of guesses used then return to a main menu or quit based the user’s choice.

The main menu will be presented before the start of each game. The user will select from 3 possible number guessing games and will also have the option to quit. The menu should look something like this:

Super Number Guessing Game Deluxe
Please Choose:
1: Easy Guess a number between 1 and 20
1: Normal Guess a number between 1 and 100
1: Hard Guess a number between 1 and 1000
0: Exit

Note: Use a RAW String to create this menu (""" """)

# Inheritance

You will use the provided RandomNumber class as the superclass for your project:

RandomNumber.kt.txt

Note: You will need to make sure that the package name matches your project

Then create a new subclass that inherits from the RandomNumber class and name it RangedRandomNumber.

# The subclass:

Add two new Integer properties to the subclass class:

  • minimum: this field represents the lowest number the random object should create

  • maximum: this field represents the highest number the random object should create

The primary constructor will have integer arguments for both the minimum and maximum values.

Add set methods for minimum and maximum that validate the passed argument making sure they are in the correct range for the Random object (minimum should be zero or above, maximum should be one or above) and the type of Integer container you are using to store the value.

All validation errors must throw an exception with an appropriate error message.

Add an additional error message which provides a more detailed description on the error if an exception is thrown during validation.

Make sure these set methods are called when an instance of this class is created.

The superclass should be created in such a way that it cannot be inherited by another class (hint: remember this is Kotlin not Java).

Override the GenerateRandomNumber method in the superclass so that it generates a random number in the required range for your Game using the minimum and maximum fields you added.

# Program Code Structure:

You will add a new class to a newly created Kotlin console application project. The new project should be named Guessing Game and the new class should be named Game.

The Game class will contain the following properties:

  • A public Integer that is to be used to keep track of the total number of guesses a player makes in a single game.
  • A private Derived Random object (subclass) that will be used to generate the random number to be guessed.
  • A private nullable String that will be used to capture player input.
    • Only this String is to be used for player input inside the Game class.

The Game class will contain the following methods and variables:

  • An Integer to count the number of guesses which should be initialized to zero.

  • A nullable String used for input which should be initialized to empty.

  • A RangedRandomNumber object to be used to generate (pseudo) random numbers.

  • A public Display Menu method:

    • This method has no arguments and returns a Short.
    • This method displays the games main menu, captures the players menu choice and validates it:
    • First this method displays the main menu. The user will enter an integer value between 0 and 3. The users input must be a valid number in the required range, if not it will loop until a valid number is provided. This method will only return once a valid choice is made.
    • This method will use a try catch structure to make sure that the input is in the correct format.
      • The catch structure will use a when statement that will check for a NumberFormatException, an ArithmeticException and then use the else option to display the exception message if the event was not caused by these two identified exceptions.
    • Once a valid choice is made this method will return by calling the Game Setup method described below and pass the player choice as an argument.

  • A private Game Setup method:

    • This method has one argument, a short  and returns a short
    • This method will Setup the Game by deciding what range of random number is to be generated based on the player choice.

  • Internally a when statement will be used to examine all four possible cases (0 – 3).

  • If the player choice is zero this method will simply return that zero value.

  • The other three possible values will generate a random number using the following range values: 1: Generate a random number between 1 – 20 2: Generate a random number between 1 - 100 3: Generate a random number between 1 - 1000

  • The method will then return the random number that is generated

  • A public Play Game method:

    • This method has one argument, an Integer and has no return value
    • This method contains the actual game play, the number guessing game.
    • The argument contains the number to be guessed.
    • All game play takes place inside this method. The player will provide a guess and the game will give feedback, based on if the guess is too high or too low or correct.
    • This method will use a try catch structure to make sure that the input is in the correct format.
      • The catch structure will use a when statement that will check for a NumberFormatException, an ArithmeticException and then use the else option to display the exception message if the event was not caused by these two identified exceptions.

  • With every guess the data member that contains the total number of guesses made will be incremented.

  • Once the correct guess is made this data member will contain the total number of guesses made in the current game.

  • Once the correct guess is made the method will return.

You will need to create all the required code and logic to perform this required functionality.

# The main function:

First create a new instance (object) of your Game class.

Then create a local nullable String that is to be used just for player input. This is for use in the Main function only. The Game class has a property that is to be used for input inside the Game class only.

Next the programs main function will use a do while loop to control the flow of the game. This loop will exit if the user enters either a lowercase or an uppercase n. Any other character will cause the loop to execute over again.

Inside main, the do while loop will first call the game object method that displays the menu (as described above).

The Display Menu method displays the main menu.

The player will enter an integer value between 0 and 3. The users input must be a valid number in the required range, if not it will loop until a valid number is provided. The method will only return once a valid choice is made.

The Display Menu method must the return the players choice (0 – 3) to another game object method: Game Setup.

Also note: This method must use a raw String to display the actual menu

If a zero is returned from this method the program must exit the do while loop then, display an exit message then immediately terminate.

If the return value is not zero then the program will call the game object method that plays the game, passing the generated random number (the number the player needs to guess).

Once this method has returned the player has successfully guessed the random number.

At this point (the end of each game) a congratulations message will be displayed (from the Main method) and the total number of guesses will be shown to the player along with a message that asks if the user wants to play again. The message should accept a lowercase and uppercase “n” as no (which means the game will immediately exit), or any other key to play again. If playing again the guess counter must be set back to zero before each game.
