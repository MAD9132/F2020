# Exercise 2 - The return of the Game Character

# Details

This in class exercise will demonstrate your understanding of both Kotlin and Object Oriented Programming within the Kotlin language.

NOTE: You must be in class to complete this exercise.

You will use many OOP concepts in this exercise including inheritance.

Using Kotlin and the superclass we created together in class:

player.kt.superclass-for-exercise.txt

You will create the subclass that represent the final version of the game character previously used in the OOP course.

# The subclass:

Add 5 more properties:

  • faction (whose side or team the character is on, must have at least 2 sides)
  • strength (maximum value is 100, minimum is 0)
  • stamina (maximum value is 500, minimum is 0)
  • weapons (maximum value is 10, minimum is 1)
  • money ((maximum value is 1,000,000 , minimum is - 1,000,000 )

NOTE: Make sure you create and use constants to represent the valid values listed above

Each new property must have a setter method and that validates the values being applied to make sure they are valid.

Make sure you also use the new setter methods to validate the passed arguments passed to the class during construction and deal with all possible error situations. All validation errors must throw an exception with an detailed error message.

You should only have a primary constructor in the subclass. This  constructor must include these new properties and also the properties used in the superclass.

All 5 new properties should have default values declared in the primary constructor

Make sure you pass all the superclass properties to the superclass constructor

Override the output method so that it includes all 5 new properties and make sure the output is properly formatted.

Add a new public method in the subclass:

Add a new public method that returns the characters rank and name as a single String, for example "General Motors". Name this method appropriately.

# Main Function:

Add a main function in the subclass file then create 3 new PlayerV2 instances using the subclass:

  • one which is passed no arguments
  • one which is passed ALL the arguments
  • one which is passed just the name, rank and money arguments

Output (using the output method) all 3 PlayerV2 instances.

Also in the main function output the rank and name of all 3 PlayerV2 instances using the newly added method that returns the Rank and Name as a single string.

## Regions:

Add folding regions for the following code sections in your subclass:

  • parameters
  • constants
  • properties
  • methods
  • init method
  • main function

# Submission

During class you will build and run this application then you will show it to me running on your laptop.

After you have shown me your app running and I have recorded your name, you will submit to the LMS your subclass source code as a text file.

To do this you will copy your Kotlin subclass source code file PlayerV2.kt then rename it by adding a ".txt" extension PlayerV2.kt.txt

Note: I will demonstrate this during class.

Note: Make sure your Kotlin subclass source code file has your name and the date as a comment at the top of the file.
