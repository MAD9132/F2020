# Save and Load the three Search Options using SharedPreferences:

Use the SharedPreference class to save and load the following 3 required values correctly in MainActivity:

  • minimum number of repos
  • minimum number of followers
  • number of results per page

You should add the following 3 new String resources for the SharedPreferences keys:

<string name="repos_key">REPOS_KEY</string>
<string name="followers_key">FOLLOWERS_KEY</string>
<string name="page_size_key">PAGE_SIZE_KEY</string>
1
2
3

Your app should check for these 3 values in SharedPreferences inside the onCreate method in MainActivity.

If the value does not exist in SharedPreferences you should use a default value.

Here's an example of loading from SharedPreferences and proving a default if the value does not exist:

if(sharedPreference.contains(getString(R.string.repos_key))) {
  minReposEditText.setText(sharedPreference.getValueInt( getString(R.string.repos_key)).toString())
} else {
  minReposEditText.setText("0")
}

Code quality, comments and regions:

Source code should be professional and correctly formatted.

Add all group members full names and the date at the top of all Kotlin source code files in this project.

Add separate code regions in all Kotlin source code files in the project for properties and methods.

Also add code regions for large areas of code such as RecyclerView.

Add appropriate comments to your code for new technologies covered in this project. These comments should be written so that you can easily understand how to re-use it in another project.

Remove ALL code you have commented out or are not using in the project.

Remove ALL unused variables, methods and unnecessary null checks from your source code.

Remove ALL unused imports from your source code.

Remove ALL unreachable code from your source code.
