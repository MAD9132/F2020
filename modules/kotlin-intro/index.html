# Kotlin Intro

Kotlin is a cross-platform, statically typed, general-purpose programming language, that supports both OOP and functional design.

Kotlin is designed to interoperate fully with Java and the JVM.

Kotlin is more concise than Java reducing the amount of boilerplate code that you have write.

Kotlin is explicit which means it will only perform instructions if you directly tell it to do so. For example, unlike Java, you cannot explicitly assign an Int to a Long. Instead you need to use a conversion method that is included with all the numeric data types

Kotlin is safer than Java as it avoids NullPointerExceptions due to its support of nullability as part of its data type structure.

Note: Every variable in Kotlin is not-null by default

The Kotlin programming language is both free and open source.

It's available under the Apache 2 license and the project is hosted on Github:

https://github.com/JetBrains/kotlin (opens new window)

# Data Types:

Numeric data types in Kotlin are very similar to Java.

# Integers:

Kotlin supports 8, 16, 32 and 64 bit integers (Byte, Short, Int and Long):

  • Byte - 8 bit
  • Short- 16 bit
  • Int - 32 bit
  • Long - 64 bit

Note: All integers in Kotlin are signed

# Floating Point (decimal) numbers:

  • Float - 32 bit single-precision
  • Double - 64 bit double-precision

# Boolean:

The type Boolean in Kotlin is the same as the Java Boolean and is used to represent true or false logical values.

# Char:

Characters are represented with the type Char.

In Kotlin (unlike Java) Char types cannot be treated as numbers.

Char types are declared using single quotes.

# Strings:

Strings are immutable and are represented using the String class.

Note: All data types in Kotlin also have the standard conversion to String helper function:

  toString()

# Kotlin Naming Convention

Kotlin follows standard Java naming conventions:

Names of packages are lower case and do not use underscores

Names of classes and objects start with an upper case letter and use camelCase

Names of functions, properties and local variables start with a lower case letter and use camelCase with no underscores

Names of constants should use uppercase underscore_separated names

The official list of Kotlin coding conventions:

https://kotlinlang.org/docs/reference/coding-conventions.html (opens new window)

# Creating Variables in Kotlin

Kotlin variables can be mutable or immutable.

Variables are declared using two keywords: val (immutable) or var (mutable).

When using Kotlin immutable values ​​are preferred when possible.

var x: Int = 42 // creates a mutable Int variable called x which contains 42
val name: String = "Tony" // creates an immutable variable called name which contains "Tony"
1
2

Note how the variable name comes before the data type in Kotlin.

First you declare the variable using val or var then a space and then the variable name, this is followed by a colon, then a space then finally the data type.

Kotlin utilizes type inference and prefers that you don't specify the type of variable you are declaring, as long as the compiler can infer it.

var x = 42 // creates a mutable Integer variable called x which contains 42, the type is inferred to be Int
val name = "Tony" // creates an immutable variable called name which contains "Tony"
1
2

Note that the code above doesn't contain any semicolons as Kotlin does not require semicolons.

# Arrays

The Array is the most basic Kotlin collection type and supports the following features:

  • Array stores objects of the same type and its sized is fixed
  • Standard array indexing is used to identify a position in an array
  • The first element is zero
  • The last element index is equal to the array’s size minus one
  • The array size is immutable
  • The array elements are mutable

To create an array call arrayOf(passed-values).

The Kotlin compiler knows what type of array it is creating based on the type of the values passed.

The following code creates an array of integers with six elements:

val arrayOfIntegers = arrayOf(0, 2, 4, 6, 8, 10)
1

There several ways to iterate through an array:

Using a for loop:

for (number in arrayOfIntegers) {
    println(number)
}
1
2
3

Using the built-in array forEach method:

arrayOfIntegers.forEach { number -> println(number) }
1

Use array indexing to change element values:

arrayOfIntegers[0] = 42
1

Note: If you try to access an element with an index that is larger than the last index (out of bounds), or use a negative index, an ArrayIndexOutOfBoundsException will be thrown.

You can use array.size to determine the size of an array and array.lastIndex to get the value of the last index (size - 1).

# Where's My Void

Having trouble finding void?

In Kotlin everything is an object so there is no void.

Void has been replaced with the Unit object.

The Unit object represents an empty value.

The usage is similar to void therefore the Unit keyword can usually be omitted as it is not required.

Note that ALL functions in Kotlin return a value, therefore a Kotlin function that has no return type will return the object Unit

# Nullable Types in Kotlin

Kotlin supports nullability with its data types.

When creating a Kotlin variable you declare whether a variable can hold a null value or not.

This has the advantage of allowing the compiler to detect possible NullPointerException errors at compile time, which reduces the occurrence of these errors at run time.

All variables in Kotlin are non-nullable by default.

If you attempt to assign a null value to a standard Kotlin variable the compiler will throw an error:

var cheese: String = "Cheddar"
cheese = null // This produces a compilation Error
1
2

To support null values in variables Kotlin requires you to declare them as nullable (they are allowed to contain null) when they are created.

This is accomplished by simply appending a question mark to the type declaration:

var nullableCheese: String? = "Cheddar"
nullableCheese = null // this is allowed because nullableCheese is allowed to contain null
1
2

NullPointerException errors commonly occur when trying to access a property or call a method on a variable which is set to null.

Kotlin prevents property access and method calls on nullable variables and preventing many NullPointerExceptions errors.

The following code is allowed because Kotlin knows that the variable cheese can never be null:

val length = cheese.length
val uppercaseCheese = cheese.toUpperCase()
1
2

However the same code fails when using a nullable variable:

var cheese: String? = "Cheddar"

val length = nullableCheese.length // This produces a compilation Error
val uppercaseCheese = nullableCheese.toUpperCase() // Another compilation Error
1
2
3
4

The Kotlin compiler knows, at compile time, which variables can contain null. This allows Kotlin to detect and prevent statements that may result in a NullPointerException.

Not null assertion : !! Operator

The not null assertion operator !! first converts a nullable type into a non-null type and then throws a NullPointerException if the nullable type contains a null value:

val cheese: String? = null
cheese!!.toUpperCase() // Throws a NullPointerException
1
2

::: Warning As the not null assertion operator circumvents the nullability technology built into Kotlin, it should only be used when absolutely necessary. :::

# Working with Nullable Types

There are several ways to safely work with nullable types in Kotlin

Using a traditional null check:

val cheese: String? = "Swiss"

if(cheese != null) {
    println(cheese.toUpperCase())
} else {
    println("Your cheese is null!")
}
1
2
3
4
5
6
7

Note: The compiler knows that the null check has been performed and allows subsequent calls (such as toUpperCase()) in the if branch of the condition statement.

Safe call operator: ?.

A null comparison as shown above is simple but requires extra code (the condition statement)

To simplify this code Kotlin provides the Safe call operator ?.

The Safe call operator allows for more concise code by combining both the null-check and a method call into a single expression:

cheese?.toUpperCase()
1

The statement above is equivalent to:

if(cheese != null) {
    println(cheese.toUpperCase())
}
1
2
3

To safely output to the console the String variable cheese in uppercase and also its length:

val cheese: String? = null

println(cheese?.toUpperCase())
println(cheese?.length)
1
2
3
4

The code above will output null for both statements as that is the value of the cheese String variable

console output:

null
null
1
2

You can also create code that only executes an operation if the variable is not null, by using both the Safe call operator with the let extension function:

val cheese: String? = null

cheese?.let { println(it.toUpperCase()) }
cheese?.let { println(it.length) }
1
2
3
4

The expression inside the let function will only execute only if the variable cheese is not null.

console output:
1

nothing is output

Safe operator calls can be chained to simplify code:

val studentEmail: String? = number?.name?.email
1

In this code the variable studentEmail will be null if either number, name or email are null.

The Elvis operator: ?:

The Elvis operator ?: is similar to the Safe call operator with an additional feature, it provides a default value when the variable is null.

The Elvis operator takes two values, it returns the first value if the variable is not null, else, it returns the second value:

val myCheese = cheese ?: "Cheddar"
1

The code above is equivalent to:

val myCheese = if(cheese != null) cheese else "Cheddar"
1

Note: Even when using shorthand in Kotlin (above) the Elvis operator provides a less verbose option

The Elvis operator can be used with Safe call operator to provide a default value:

val length = cheese?.length ?: -1

val studentEmail = number?.name?.email ?: "No Email Address"
1
2
3

The Elvis operator can also be used to throw an exception:

val myCheese = cheese ?: throw IllegalArgumentException("Your cheese cannot be null")
1

# Data Type Conversion

Kotlin (unlike Java) does not support implicit conversion from smaller data types to larger data types.

For example, you cannot assign an Int to a Long or a Double

In addition to the standard toString() helper function all Kotlin numeric data types have conversion helper functions that are used to convert from one data type to another:

  • toByte()
  • toShort()
  • toInt()
  • toLong()
  • toFLoat()
  • toDouble()
  • toChar()

# Lists

Lists are dynamic arrays.

List collections are available in two different versions: mutable and immutable, therefore:

Lists can have either a mutable or immutable size Lists can have either mutable or immutable elements

Creating immutable Lists:

val listOfIntegers = listOf(0, 2, 4, 6, 8, 10)
1

Note: You cannot change element values in an immutable List:

listOfIntegers[0] = 42 // this will cause an error!
1

Creating mutable lists:

val listOfIntegers = mutableListOf(0, 2, 4, 6, 8, 10)
1

Mutable Lists are dynamic as they allow adding, removing and inserting elements.

Mutable Lists allow for element content to change:

val listOfIntegers = mutableListOf(0, 2, 4, 6, 8, 10)
listOfIntegers[0] = 42 // changes the element value
listOfIntegers.removeAt(1) // removes the element
listOfIntegers.add(index = 2, element = 13) // insert at index 2
listOfIntegers.remove(13) // removes the element with the value 13
listOfIntegers[1] = 10
listOfIntegers[2] = 10

# Sets

Sets are a variation of the List collection where each of the elements is unique. They are commonly used when duplicate values are not allowed, for example a Set of student numbers.

Sets can have either a mutable or immutable size Sets can have either mutable or immutable elements Duplicate elements (values) are not allowed

Creating immutable Sets:

val setOfIntegers = setOf(0, 2, 4, 6, 8, 10)
1

Note: You cannot change element values in an immutable Set:

setOfIntegers[0] = 42 // this will cause an error!
1

Creating mutable Sets:

val setOfIntegers = mutableSetOf(0, 2, 4, 6, 8, 10)
1

Mutable Sets are dynamic as they allow adding, removing and inserting elements.

Note: Sets use the hashCode() method internally to filter out duplicates. If the Set is mutable adding a duplicate element to it replaces the original item with the new one.

Note: The hashCode() method can be overridden to change the default behaviour.

Note: A Set is an unordered collection: Sets do not have indexes (indices), because all values are unique.

When using Sets it does not matter what order is used to insert values:

setOfIntegers.remove(10) // this removes the element that contains the number 10
1
